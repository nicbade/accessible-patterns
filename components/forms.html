Using a screen reader in Browse mode (Forms)

In Browse (Virtual) mode, a screen reader lets you read a page with the arrow keys and jump by element type (headings, links, form fields). When you focus an input, many readers switch to Forms/Focus mode automatically so you can type. You can also toggle modes manually.

Tip: stay in Browse mode to skim and locate fields, then switch to Forms/Focus mode only when you’re ready to edit. Press Esc to exit Forms/Focus mode quickly.

Keyboard-only (no screen reader)

  • Use Tab/Shift+Tab to move between fields and buttons.
  • Use Space to check/uncheck checkboxes; Arrow keys to change radios or move in selects.
  • Use Enter to activate the focused button (e.g., Submit).

JAWS / NVDA in Browse mode

  • Move around in Browse mode: Use the Arrow keys to read; use quick-nav letters:
    • H/Shift+H — next/previous heading
    • F/Shift+F — next/previous form field
    • E/Shift+E — next/previous edit (text) field
    • B/Shift+B — next/previous button
  • Enter Forms/Focus mode to type:
    • NVDA: Press NVDA+Space to toggle Browse/Focus mode manually (auto on most fields).
    • JAWS: Toggle Virtual PC Cursor with Insert+Z if a control won’t activate in Browse mode.
    • Press Esc to leave Forms/Focus mode and return to Browse mode.
  • Jump to a list of elements:
    • NVDA: NVDA+F7 opens the Elements List (filter by Form fields, Buttons, etc.).
    • JAWS: Insert+F5 lists form fields; Insert+F6 lists headings.
  • While in a table inside a form: Use Ctrl+Alt+Arrow keys to move cell-by-cell; headers are announced with cells when marked up correctly.
