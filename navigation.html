Keyboard navigation

Core movement

  • Tab moves forward to the next focusable element (links, buttons, inputs, controls).
  • Shift+Tab moves backward.
  • Enter / Space activates the focused control (follow a link, press a button, toggle a checkbox).
  • Esc closes/cancels open UI when supported (menus, dialogs, tooltips).
  • Arrow keys move within certain widgets (e.g., tabs, radio groups, menus, carousels).
  • The focused element must show a visible focus ring; if you can’t see it, that’s a bug.

Tips & expectations

  • Use the **Skip link** (first Tab stop) to jump directly to main content.
  • Tab order should follow the visual order; unexpected jumps indicate a focus-order issue.
  • You should never get trapped: Esc closes popups; Tab/Shift+Tab move focus.
  • macOS: enable full keyboard navigation (System Settings → Keyboard → “Keyboard Navigation”) and in Safari, “Press Tab to highlight each item on a webpage”.

Quick test checklist

  • Press Tab on the homepage: a **Skip to main content** link appears, then header links, then main controls in logical order.
  • Every focused item shows a **visible focus ring**.
  • Enter/Space activates focused controls (links, buttons, toggles).
  • Open a mobile menu or modal: focus is **trapped** inside with Tab/Shift+Tab; Esc closes and returns focus to the trigger.
  • On forms: Space toggles checkboxes; Arrow keys change radios; Enter submits only when a submit button is focused.
